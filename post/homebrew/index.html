1、Homebrew简介#

官方网站：

https://brew.sh

官方文档：

https://docs.brew.sh

Homebrew是macOS平台下的一款包管理工具，用户仅需通过一条简单的指令就可实现软件包管理，极大程度节省了构建环境所需的时间。

2、Homebrew安装#

安装文档：

https://docs.brew.sh/Installation#1

安装要求：

  • 64位Intel CPU或Apple Silicon CPU
  • macOS Mojave（10.14）或更高版本
  • Xcode命令行工具：xcode-select --install
  • 支持shell

在终端输入如下命令安装Homebrew：

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

卸载Homebrew：

cd `brew --prefix`
rm -rf Cellar
brew prune
rm `git ls-files`
rm -r Library/Homebrew Library/Aliases Library/Formula Library/Contributions
rm -rf .git
rm -rf ~/Library/Caches/Homebrew

3、Homebrew基本使用#

安装包：

brew install <package>

// 安装node
brew install node

卸载包：

brew uninstall <package>

查询包：

brew search <package>

查看已安装包列表：

brew list

查看包信息：

brew info <package>

更新Homebrew：

brew update

查看Homebrew版本：

brew -v

查看帮助信息：

brew -h

4、注意事项#

macOS X 10.11版本之后，/usr/local/等系统目录下的文件读写需要root权限，而Homebrew在安装时如果未指定安装路径，则会默认安装在需要root权限的目录下，在执行特定指令时需要添加sudo前缀。如对Homebrew进行升级：

sudo brew update

为了解决这个问题，我们可以通过以下两种方式进行解决：

方式一#

对/usr/local/目录下的读写文件进行root授权：

sudo chown -R <user> /usr/local

此处的<user>为用户名，具体命令如下：

sudo chown -R 0opsdc /usr/local

方式二（推荐）#

指定Homebrew安装路径为不需要root用户授权的目录：

<install path> -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
